What

Curies implements the CURIE syntax for expressing Compact URIs. See http://www.w3.org/TR/curie/ for more information.

Installing

sudo gem install curies

The basics

In a nutshell, a CURIE is a compact way of representing a URI. For example, suppose you’ve got an XML document in which you’ll be writing many attributes of the form “http://www.amazon.com/?isbn=123478753”. With a CURIE, you can represent those as “[amazon:123478753]” where “amazon” is a registered prefix in your document.

This library is small, but useful if you’re writing a parser for a language that uses CURIEs. I wrote it because I’m implementing an RDF/JSON library and, because the RDF/JSON spec isn’t finalized yet, I’ve taken the liberty of using CURIEs rather than QNAMEs.

Demonstration of usage

require 'curies'
c = Curie.new("foaf", "person")

c.to_s
=>"[foaf:person]"

Curie.parse "[foaf:name]"
"http://xmlns.com/foaf/0.1/person"

Curie.parse "[foo:bar]"
"RuntimeError: Sorry, no namespace or prefix registered for curies with the prefix foo"
Curie.add_prefixes! :foo => "http://www.example.com/ns/foo/"
=> {"foo"=>"http://www.example.com/ns/foo/", "foaf"=>"http://xmlns.com/foaf/0.1/"}
Curie.parse "[foo:bar]"
=> "http://www.example.com/ns/foo/bar"
